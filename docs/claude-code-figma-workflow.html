1. Le concept : code → canvas → code

Le blog Figma du 17 février 2026 annonce une vision : capturer des interfaces construites en code et les convertir en frames Figma éditables. Le code est puissant pour converger (build, test, itérer). Le canvas Figma est puissant pour diverger (explorer, comparer, collaborer).

La boucle complète : Claude Code génère du HTML/CSS → on capture en Figma → on itère visuellement sur le canvas → le MCP Figma ramène le design dans le code. C'est un roundtrip bidirectionnel.

Mais au-delà de la capture d'écran, il existe un workflow plus puissant : piloter Figma directement depuis Claude Code via MCP — créer des frames, appliquer des polices, construire des composants, importer des images, le tout par commandes.

Claude Code ←→ MCP Server ←→ WebSocket / API ←→ Figma Desktop

2. Architecture : 2 MCP, 2 plugins, 1 workflow

Deux MCP complémentaires permettent de couvrir tous les cas d'usage :

claude-talk-to-figma figma-console-mcp
Rôle principal Création et modification temps réel Extraction, debug, execute arbitraire
Connexion WebSocket + Plugin (channel ID) API Token + Desktop Bridge plugin
Plugin Figma Claude MCP Plugin Figma Desktop Bridge
Port 3055 (WebSocket) 9223–9232 (multi-instance)
Nb d'outils ~25 outils spécialisés ~56 outils + figma_execute
Import images Non Oui (via figma_execute)
Auteur arinspunk southleft
Pourquoi les deux ? claude-talk-to-figma excelle en création rapide (frames, textes, rectangles, auto-layout). figma-console-mcp excelle en opérations avancées (execute JS arbitraire, image fills, variables, design tokens, screenshots). Ensemble, ils couvrent 100% des besoins.

3. Setup 1 — Claude Talk to Figma

Prérequis

  • Bun installé (curl -fsSL https://bun.sh/install | bash)
  • Figma Desktop (pas le navigateur)

Installation pas à pas

1
Cloner et builder le repo 
git clone https://github.com/arinspunk/claude-talk-to-figma-mcp.git ~/claude-talk-to-figma-mcp
cd ~/claude-talk-to-figma-mcp
bun install
bun run build
2
Lancer le serveur WebSocket 
cd ~/claude-talk-to-figma-mcp && bun run dist/socket.js

Le serveur écoute sur ws://localhost:3055. Vérifier : http://localhost:3055/status

3
Importer le plugin dans Figma

Dans Figma Desktop :

  1. Menu hamburger (☰) → Plugins → Development → Import plugin from manifest…
  2. Sélectionner le fichier :
    ~/claude-talk-to-figma-mcp/src/claude_mcp_plugin/manifest.json
4
Connecter le plugin

Dans votre fichier Figma :

  1. Plugins → Development → Claude MCP Plugin
  2. Cliquer Connect
  3. Noter le channel ID affiché (ex: 3qbur6w5)
5
Dans Claude Code : rejoindre le channel

Claude utilise l'outil join_channel avec le channel ID. À partir de là, tous les outils de création sont disponibles.

Manifest du plugin Claude MCP

manifest.json 
{
  "name": "Claude MCP Plugin",
  "id": "claude-mcp-plugin",
  "api": "1.0.0",
  "main": "code.js",
  "ui": "ui.html",
  "editorType": ["figma", "figjam", "dev"],
  "permissions": ["teamlibrary"],
  "networkAccess": {
    "allowedDomains": ["https://google.com"],
    "devAllowedDomains": [
      "http://localhost:3055",
      "ws://localhost:3055"
    ]
  },
  "documentAccess": "dynamic-page",
  "enableProposedApi": true,
  "enablePrivatePluginApi": true
}

Le système de channels

Le WebSocket server gère des channels nommés. Quand le plugin Figma se connecte, il génère un channel ID aléatoire (ex: 3qbur6w5). Le MCP server Claude rejoint ce même channel. Tous les messages transitent par le channel — c'est un système pub/sub simplifié.

Claude MCP → join("3qbur6w5") → WebSocket :3055 ← join("3qbur6w5") ← Figma Plugin

4. Setup 2 — Figma Console MCP

Prérequis

  • Node.js ≥ 18
  • Figma Desktop
  • Un Personal Access Token Figma

Obtenir le token Figma

  1. Aller sur figma.com/developers/api#access-tokens
  2. Créer un nouveau token personnel
  3. L'exporter dans votre shell : 
    export FIGMA_API_KEY="figd_votre_token_ici"

Installation

1
Le MCP s'installe via npx (pas de clone nécessaire) 
# Dans la config Claude, le MCP est défini comme :
"figma-console-mcp": {
  "command": "npx",
  "args": ["figma-console-mcp"]
}
2
Importer le plugin Desktop Bridge dans Figma

Dans Figma Desktop :

  1. Menu (☰) → Plugins → Development → Import plugin from manifest…
  2. Naviguer vers le manifest du Desktop Bridge :
    ~/.npm/_npx/<hash>/node_modules/figma-console-mcp/figma-desktop-bridge/manifest.json
Trouver le chemin exact : Le hash npx change selon votre système. Utilisez : 
find ~/.npm/_npx -name "manifest.json" -path "*desktop-bridge*"
3
Lancer le plugin Desktop Bridge dans Figma
  1. Plugins → Development → Figma Desktop Bridge
  2. Le plugin se connecte automatiquement sur ws://localhost:9223
  3. Status "Connected" = prêt

Manifest du Desktop Bridge

manifest.json 
{
  "name": "Figma Desktop Bridge",
  "id": "figma-desktop-bridge-mcp",
  "api": "1.0.0",
  "main": "code.js",
  "ui": "ui.html",
  "editorType": ["figma", "dev"],
  "capabilities": ["inspect"],
  "enablePrivatePluginApi": true,
  "permissions": ["teamlibrary"],
  "documentAccess": "dynamic-page",
  "networkAccess": {
    "allowedDomains": [
      "ws://localhost:9223",
      "ws://localhost:9224",
      "... (ports 9223–9232)"
    ]
  }
}

L'outil secret : figma_execute

Le Desktop Bridge déverrouille l'outil figma_execute — il permet d'exécuter du JavaScript arbitraire dans le contexte plugin Figma. C'est l'accès complet à la Figma Plugin API. Vous pouvez créer des images, manipuler des nodes, tout ce que l'API plugin permet.

C'est la clé pour l'import d'images. Ni claude-talk-to-figma ni les outils standard de figma-console-mcp ne peuvent importer des images. Seul figma_execute avec du code Plugin API peut le faire (via figma.createImage()).

5. Le workflow combiné en pratique

Avant de commencer

Dans votre terminal, 3 éléments doivent tourner :

  1. WebSocket server : cd ~/claude-talk-to-figma-mcp && bun run dist/socket.js
  2. Figma Desktop ouvert avec les 2 plugins connectés :
    • Claude MCP Plugin (channel ID noté)
    • Figma Desktop Bridge (status "Connected")
  3. Claude Code lancé dans votre projet

Séquence typique

Phase 1 : Scaffolding (claude-talk-to-figma)

Création rapide de la structure :

// Claude rejoint le channel
→ join_channel("3qbur6w5")

// Création des frames, rectangles, textes
→ create_frame({ name: "Desktop — Accueil", width: 1440, height: 1800, ... })
→ create_rectangle({ name: "grid__item--a", parentId: "...", ... })
→ create_text({ text: "Techniques graphiques", parentId: "...", ... })

// Auto-layout
→ set_auto_layout({ nodeId: "...", layoutMode: "VERTICAL", itemSpacing: 12, ... })

// Polices
→ load_font_async({ family: "GTWalsheim-Light", style: "Regular" })
→ set_font_name({ nodeId: "...", family: "GTWalsheim-Light" })

// Composants
→ create_component_from_node({ nodeId: "...", name: "sidebar" })

Phase 2 : Enrichissement (figma-console-mcp)

Opérations avancées via figma_execute :

// Importer une image dans un rectangle
→ figma_execute({
    code: `
      const imageBytes = await fetch('...').then(r => r.arrayBuffer());
      const image = figma.createImage(new Uint8Array(imageBytes));
      const node = figma.getNodeById('4:295');
      node.fills = [{ type: 'IMAGE', imageHash: image.hash, scaleMode: 'FILL' }];
    `
  })

// Screenshot pour vérification
→ figma_take_screenshot()

// Extraction de design tokens
→ figma_get_variables()
→ figma_get_styles()

Phase 3 : Roundtrip vers le code

Avec le Figma MCP Server officiel (annoncé dans le blog post), on peut ramener le design Figma dans le code via un prompt + lien de frame. C'est la boucle complète : Code → Figma → Code.

7. Import d'images programmatique

C'est le cas d'usage qui nécessite les deux MCP ensemble. claude-talk-to-figma crée les frames et placeholders. figma-console-mcp (via figma_execute) remplit les placeholders avec de vraies images.

Méthode : image fill via figma_execute

La Plugin API Figma expose figma.createImage(bytes) pour créer une image à partir de bytes bruts (Uint8Array). On peut ensuite l'appliquer comme fill sur n'importe quel nœud.

Exemple : remplir un rectangle avec une image encodée en base64 
figma_execute({
  code: `
    // Décoder le base64 en bytes
    const base64 = "iVBORw0KGgo..."; // image en base64
    const raw = atob(base64);
    const bytes = new Uint8Array(raw.length);
    for (let i = 0; i < raw.length; i++) {
      bytes[i] = raw.charCodeAt(i);
    }

    // Créer l'image dans Figma
    const image = figma.createImage(bytes);

    // Appliquer comme fill sur le nœud cible
    const node = figma.getNodeById('4:295');
    node.fills = [{
      type: 'IMAGE',
      imageHash: image.hash,
      scaleMode: 'FILL'
    }];

    return { success: true, nodeId: '4:295', imageHash: image.hash };
  `
})
Taille des images : Les images sont transmises en base64 via le MCP, ce qui augmente la taille de ~33%. Pour des images volumineuses (>5 MB), il est préférable de les redimensionner/compresser côté Claude Code avant envoi, ou d'utiliser fetch() depuis une URL locale si le plugin le permet.

Workflow complet pour remplir une grille

  1. Claude Code lit les images du dossier assets/images/
  2. Les encode en base64 (ou les redimensionne si trop grandes)
  3. Appelle figma_execute pour chaque image avec le nodeId du placeholder
  4. Prend un screenshot pour vérifier le résultat

8. Limitations et contournements

Limitation Détail Contournement
Polices variables Les variable fonts (.ttf VF) provoquent un timeout systématique via load_font_async Appliquer manuellement dans Figma, ou utiliser figma_execute avec loadFontAsync()
Import images claude-talk-to-figma ne peut pas importer d'images Utiliser figma_execute via figma-console-mcp
CSS Grid 2D L'auto-layout Figma est 1D (flexbox), pas 2D (grid) Auto-layout sur les sous-conteneurs, positionnement statique pour la grille
Appels parallèles Trop d'appels MCP simultanés (>4) causent des timeouts WebSocket Limiter à 2-3 appels en parallèle, alterner scans/modifications
Variant sets combineAsVariants() timeout quand composants sont intégrés dans un layout Nommer les composants Property=Value, Figma les regroupe dans Assets
Connexion instable Le WebSocket peut décrocher après beaucoup d'opérations Reconnecter le plugin → nouveau channel ID → join_channel

    "claude-talk-to-figma": {

      "command": "npx",

export FIGMA_API_KEY="figd_votre_personal_access_token"
